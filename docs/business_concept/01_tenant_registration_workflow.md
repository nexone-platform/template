# สรุปขั้นตอนการทำงาน การลงทะเบียนใช้งานสำหรับลูกค้าใหม่ (New Tenant Registration Workflow)

กระบวนการการลงทะเบียนผู้ใช้งานใหม่ (Tenant Provisioning) ของระบบ NexCore ERP ได้ถูกออกแบบมาเพื่อรองรับสถาปัตยกรรมแบบ Multi-tenant อย่างสมบูรณ์แบบ ทั้งในส่วนของการติดต่อกับผู้ใช้งาน (Frontend) และระบบจัดการข้อมูลเบื้องหลัง (Backend) โดยมีรายละเอียดขั้นตอนดังนี้

## 1. การกรอกข้อมูลและตรวจสอบความถูกต้อง (Frontend UI)
ลูกค้าที่ต้องการใช้งานระบบจะต้องกรอกข้อมูลผ่านหน้า Registration Wizard ซึ่งจะมีการตรวจสอบความถูกต้องของข้อมูล (Validation) แบบ Real-time:
*   **Business Profile:** ลูกค้ากรอกข้อมูลพื้นฐานของบริษัท เช่น คำนำหน้านิติบุคคล, ตัวย่อบริษัท, ชื่อบริษัท (รองรับการตรวจจับภาษาไทยและอังกฤษ), เลขประจำตัวผู้เสียภาษี (ตรวจสอบให้เป็นตัวเลขเท่านั้น), ที่อยู่, และจำนวนพนักงาน
*   **Business Type:** ลูกค้าเลือกกลุ่มธุรกิจหลัก (เช่น ขนส่ง, ซื้อมาขายไป, ผลิต) และประเภทย่อย ระบบจะคำนวณสิทธิ์การใช้งานแอปพลิเคชัน (Enabled Apps) ให้โดยอัตโนมัติตามกลุ่มธุรกิจที่เลือก
*   **Admin Profile:** ลูกค้ากรอกข้อมูลสำหรับผู้ดูแลระบบ (Admin) คนแรกของบริษัท (ชื่อ, เบอร์โทรศัพท์, อีเมล) พร้อมทั้งตั้งรหัสผ่านที่รัดกุม

## 2. การส่งข้อมูลเพื่อลงทะเบียน (API Submission)
*   เมื่อผู้ใช้งานกดยืนยัน ระบบ Frontend จะส่งข้อมูลทั้งหมดในรูปแบบ JSON ไปยัง API Endpoint `POST /api/registration`
*   ระบบ Backend (`nex-core-api`) ทำการรับข้อมูล และทำการเข้ารหัสผ่าน (Password Hashing) ของ Admin เพื่อความปลอดภัยของข้อมูล

## 3. การสร้างข้อมูล Tenant เบื้องต้น (Backend Initial Persistence)
*   **Generate Database Name:** ระบบจะนำชื่อบริษัทที่ลูกค้ากรอกมาแปลงเป็นชื่อ Database สำหรับลูกค้าที่ปลอดภัยและไม่ซ้ำซ้อนกัน (เช่น `nex_testcompany_ab12`)
*   **Save Registration:** บันทึกประวัติการลงทะเบียนทั้งหมดลงในตารางกลาง (`tenant_registrations` ของ Database หลัก) และกำหนดสถานะเริ่มต้นของการประมวลผลเป็น **`provisioning`** (กำลังดำเนินการ)
*   **Return Response:** Backend จะตอบกลับ (Return) ผลลัพธ์พร้อมกับ Registration ID กลับไปยัง Frontend ทันที เพื่อให้หน้าจอฝั่งลูกค้าแสดงสถานะ Loading โดยไม่ต้องรอให้สร้างฐานข้อมูลเสร็จ ซึ่งจะช่วยลดปัญหา Timeout และทำให้ UI ตอบสนองได้ดี

## 4. กระบวนการสร้างฐานข้อมูลส่วนตัว (Background Provisioning)
กระบวนการนี้จะทำงานอยู่เบื้องหลัง (Asynchronous Process) โดยเป็นการสร้าง Database ใหม่แยกแต่ละลูกค้า (Database-per-Tenant) แบ่งออกเป็น 5 ขั้นตอนย่อย:
1.  **Create Database:** ระบบรันคำสั่ง `CREATE DATABASE` เพื่อสร้างก้อนฐานข้อมูลก้อนใหม่แยกสำหรับลูกค้ารายนี้โดยเฉพาะ เพื่อแยกพื้นที่เก็บข้อมูลของลูกค้ารายนี้ออกจากลูกค้ารายอื่น ๆ แบบ 100%
2.  **Connect to New Database:** ระบบสลับการเชื่อมต่อเข้าไปยังก้อน Database ที่พึ่งสร้างใหม่ เพื่อเริ่มจัดการโครงสร้างภายใน
3.  **Create Schemas:** ทำการสร้าง Schema ของทุกแอปพลิเคชันหรือโมดูลที่เตรียมไว้ (เช่น `nex_core`, `nex_speed`, `nex_finance`, ฯลฯ) ภายในก้อน Database ของลูกค้า
4.  **Execute Table Scripts:** ระบบจะทำการดึงไฟล์ฐานข้อมูลหลัก (`03_create_schema_tables.sql`) มารันในก้อน Database ใหม่ ทำให้ตารางของทุก Modules ถูกสร้างขึ้นอย่างถูกต้องภายใต้ Schema ของแต่ละแอป
5.  **Setup Admin User:** ระบบทำการลบ Account แอดมินระบบส่วนกลางออก และสร้าง Account **Admin ตัวจริงของลูกค้า** ขึ้นมาใหม่ ภายในตาราง `nex_core.users` ด้วยอีเมลและรหัสผ่านจากขั้นตอนการลงทะเบียน พร้อมกำหนดสิทธิ์ระดับผู้ดูแล (Role ID 1)
6.  **Update Status:** หากการสร้างโครงสร้างฐานข้อมูลทุกอย่างเสร็จสมบูรณ์ ระบบจะเปลี่ยนสถานะในตารางกลางให้เป็น **`completed`** (หากเกิดข้อผิดพลาด ระบบจะเปลี่ยนสถานะเป็น `failed` พร้อมเก็บ Error Log ไว้)

## 5. การแสดงผลความสำเร็จให้ผู้ใช้งานทราบ (Frontend Polling & Completion)
*   ในขณะที่ขั้นตอนที่ 4 กำลังประมวลผลอยู่เบื้องหลัง ฝั่ง Frontend จะทำการร้องขอสถานะ (Polling) เป็นระยะๆ ผ่าน API `GET /api/registration/status/:id`
*   หน้าจอของลูกค้าจะแสดง Animation การตั้งค่าระบบ เพื่อให้ลูกค้าทราบว่าระบบกำลังทำงานอยู่
*   เมื่อสถานะถูกเปลี่ยนเป็น **`completed`** หน้าต่างจะถูกเปลี่ยนไปเป็น "ลงทะเบียนสำเร็จ" โดยแสดงปุ่มให้ลูกค้าสามารถคลิกเพื่อเข้าสู่หน้า Login และเริ่มต้นการใช้งานระบบ ERP ด้วยพื้นที่ฐานข้อมูลของตัวเองได้อย่างสมบูรณ์
